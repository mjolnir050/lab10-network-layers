Theoretical Part

TCP (Transmission Control Protocol)

TCP is a connection-oriented protocol that ensures reliable data transmission between devices. It is widely used for applications like web browsing, email, and file transfers.

  • Parameters: Sequence number, acknowledgment number, window size, etc.
  • Features: Reliable delivery, error checking, flow control.

UDP (User Datagram Protocol)

UDP is a connectionless protocol designed for low-latency communication where reliability is not critical, such as video streaming or DNS queries.

  • Parameters: Source port, destination port, length, checksum.
  • Features: Fast transmission, no guarantee of delivery.

TCP Connection Process

  1. Three-way Handshake:
    • SYN: Client sends SYN to initiate a connection.
    • SYN-ACK: Server acknowledges with SYN-ACK.
    • ACK: Client responds with ACK, establishing the connection.
  2. Connection Termination:
    • FIN: One side sends a FIN to close the connection.
    • ACK: The other side acknowledges with ACK.

Ports

Ports are numerical identifiers used to distinguish different services on a host. For example, HTTP uses port 80, and HTTPS uses port 443.

Practical Part

Task 1: Analyze Ports with netstat and Wireshark

Steps:

  1. Open a browser and visit www.google.com.
  2. Run netstat: 
    netstat -an

    Identify local and remote ports used by the browser.

  3. Capture packets in Wireshark:
    • Start Wireshark and select your active network interface.
    • Use a filter like tcp.port == 443 to see HTTPS traffic.
    • Analyze the captured packets to identify local and remote ports.
  4. Close the browser and check the connection state using netstat.

Task 2: Transfer File Using FTP

  1. Install an FTP client (e.g., FileZilla) if not already installed.
  2. Connect to studife.it.p.lodz.pl using your credentials.
  3. Upload a test file (e.g., test.txt).
  4. Run netstat and use Wireshark to identify ports during the file transfer (typically port 21 for FTP).
  5. End the connection and verify port state with netstat.

Task 3: Transfer File Using SCP/SFTP

  1. Open a terminal or an SFTP client.
  2. Run the command: 
    scp test.txt username@studife.it.p.lodz.pl:/path/to/destination

    Replace username with your login and /path/to/destination with the target path.

  3. Monitor ports using netstat and Wireshark during the transfer (SFTP typically uses port 22).
  4. Close the connection and check port states.

Task 4: Analyze DNS Protocol

  1. Start Wireshark and filter DNS traffic: 
    dns
  2. Visit a website (e.g., www.google.com).
  3. Observe DNS queries and responses. Note that DNS typically uses UDP port 53.
  4. Analyze packet details to confirm protocol and port usage.
